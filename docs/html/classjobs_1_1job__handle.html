Detailed Description

Represents an instance of a job that has been created by the scheduler.

Job data is owned by the scheduler, be careful accessing handles if the scheduler has been destroyed.

Sets the priority of this job.

Queues are created for each priority, if you want a job to exist in multiple queues then priorities can be combined together into a bitmask (the job will only be executed once though). Worker thread pools can also be assigned to only work on jobs with specific priorities. Combining both of these can give fine grain control over when and where individual jobs are executed.
