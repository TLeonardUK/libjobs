Detailed Description

Represents an instance of an counter that has been created by the scheduler.

Events data is owned by the scheduler, be careful accessing handles if the scheduler has been destroyed.

Counters represent an atomic integer value that can be never be negative. Counters can be added to, and removed from. If a removal would result in a negative value, the counter waits until enough has been added that the value can be removed. Counters also support the ability to get/set values directly as well as wait for specific values to be set.

The functionality of counters is straight-forward but incredibly flexible, and can be used in a multitude of ways to syncronize jobs.

Implementation wise this is fairly similar to a semaphore, just with more control over the internal signal count.
